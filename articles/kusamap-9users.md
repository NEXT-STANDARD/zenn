# はじめに

こんにちは、フェニックスです。

信じられないことが起きた。

草マップwのユーザーが**9人**になっていた。

「は？9人？少なっ」と思ったそこのあなた。
ちょっと待ってほしい。

つい先日まで、ユーザーは3人だったのだ。
私と、知り合い2人。
全員顔見知り。
見知らぬ他人はゼロ。

それが、9人。
**3倍**である。

スタートアップ界隈では「3倍成長」と言えば、投資家が目を輝かせる数字だ。
もっとも、3人が9人になっただけだが。

---

## 何が起きたのか

### Zennで自虐ブログを書き続けた

この数日間、私はZennで自虐ブログを書き続けていた。

「ユーザー2人の現実」
「売上300円（自分で課金）」
「DM機能を作ったけど全員知り合いだから意味ない」
「AIによるサクラ投稿を実装した」

恥ずかしい話ばかりだ。
普通なら隠したくなるような内容を、毎日のように公開していた。

### PVが爆発した

すると、昨日のPVが**161**になっていた。

これまでの最高は18PV。
それが一気に161。
**約9倍**である。

過去一ヶ月で最高の読者数。
自虐が人の心を掴んだのかもしれない。
人間は他人の不幸が好きなのだ。

### ユーザーが増えた

そして気づいたら、草マップwのユーザーが9人になっていた。

| 区分 | 人数 |
|------|------|
| 自分 | 1人 |
| 知り合い | 3人 |
| **見知らぬ他人** | **5人** |
| **合計** | **9人** |

見知らぬ他人が5人。
宣伝も広告も何もしていない。

考えられる流入経路は一つしかない。
**Zennのブログを読んで、興味を持ってくれた人たち**だ。

それ以外に考えられない。

---

## 9人は少ないのか

### 多くの人はこう言うだろう

「9人？少なっ」
「まだ一桁じゃん」
「話題になってから記事書けよ」

分かる。分かるよ。
世の中には何万人ものユーザーを抱えるサービスがある。
9人なんて、誤差みたいなものだ。

### しかし、考えてみてほしい

**毎日9人で飲み会を開催できるだろうか？**

「今日飲み会やるから9人集まって」と言って、9人集められる人がどれだけいる？

3人ならなんとかなるかもしれない。
5人でもギリギリいける。
でも9人となると、かなりの人脈と調整力が必要だ。

日程を合わせ、場所を決め、全員に連絡し、キャンセルが出ないように祈る。
9人の飲み会を開催するのは、それなりに大変なことなのだ。

### 家で話す相手がお母さんだけの貴方方へ

ここで、9人を「少ない」と笑う人々に問いたい。

**あなたは9人も集められるのか？**

休日に連絡を取る相手が何人いる？
「今度遊ぼう」と言って、本当に遊ぶ相手が何人いる？
家で話す相手が、お母さんだけになっていないか？

...すみません、言い過ぎました。

でも、9人のユーザーを「少ない」と切り捨てるのは、あまりにも傲慢だと思うのだ。
見知らぬ5人が、わざわざ登録してくれた。
その事実を、私は噛み締めている。

---

## ユーザーが増えたので高速化を実装

### 仕方なく改善に着手

ユーザーが増えてしまった。
3人なら許されていた「もたもた感」が、9人になると許されなくなる。

仕方なく、サービスの高速化に着手した。
「仕方なく」というのは負け惜しみで、本当は嬉しくて仕方がない。

### 5フェーズの最適化

バイブコーディングで、以下の最適化を実施した。

| Phase | 内容 |
|-------|------|
| Phase 1 | 遅延読み込み |
| Phase 2 | メインページ分割 |
| Phase 3 | API最適化 |
| Phase 4 | レンダリング最適化 |
| Phase 5 | 画像最適化 |

Claude Codeに「サービスが遅いから速くして」と伝えたら、これらの施策を提案・実装してくれた。

私がやったのは「うん、やって」と言うことだけ。
バイブコーディング、本当に便利。

### もたもたUXからの脱却

結果、もたもたしていたUXが劇的に改善した。

ページの読み込みが速くなった。
地図の表示がスムーズになった。
全体的にサクサク動くようになった。

9人のユーザーに、快適な体験を提供できるようになった。
...9人だが。

---

## 新しいフェーズへの突入

### 足し算から引き算へ

これまでの開発は、ひたすら「足し算」だった。

機能を追加する。
ページを増やす。
できることを増やす。

だが、ユーザーが増えた今、「引き算」も覚えなければならない。

不要な機能を削る。
複雑なUIをシンプルにする。
読み込みを軽くする。

新機能を追加するだけでは、サービスは重くなる一方だ。
追加と削除のバランスを取らなければならない。

### 責任の重さ

ユーザーが3人の時は、気楽だった。
何をやっても、影響を受けるのは3人だけ。
全員知り合いだから、バグがあっても「ごめん」で済む。

でも、9人になった。
見知らぬ5人がいる。
彼らに対しては、「ごめん」では済まない。

サービスを提供する責任。
ユーザーの期待に応える責任。
そういうものが、少しずつ生まれてきている。

9人で大げさな、と思うかもしれない。
でも、0人と1人の間には無限の距離があるように、
3人と9人の間にも、それなりの距離がある。

## 自虐ブログの効果

### なぜ自虐が刺さったのか

考えてみると、自虐ブログが読まれた理由は分かる気がする。

世の中には「成功者の話」が溢れている。
「100万ダウンロード達成」
「資金調達成功」
「急成長の秘訣」

そんな話を聞いても、多くの人は「すごいね」で終わる。
自分とは関係ない、遠い世界の話だから。

一方、自虐ブログはどうか。

「ユーザー3人」
「売上300円（自分で課金）」
「全然話題にならない」

これなら、共感できる。
「分かる」「自分もそう」「親近感ある」
そういう感情が生まれる。

人は、成功者より失敗者に親近感を覚えるのだ。

### 失敗を晒すマーケティング

意図したわけではないが、結果的に「失敗を晒すマーケティング」になっていた。

恥ずかしい数字を公開する。
うまくいかない現実を語る。
自分を笑いものにする。

それが、読者の心を掴んだ。
そして、一部の読者がユーザーになってくれた。

マーケティングの教科書には載っていない手法だ。
でも、効果はあった。

## 今後の目標

### 次は30人

9人になった今、次の目標は**30人**だ。

なぜ30人か。
30人いれば、小さなコミュニティが成立するからだ。

30人が毎日1投稿すれば、30件の草が生える。
24時間で消えても、常に何かしらの投稿がある状態になる。
サクラに頼らなくても、地図が賑わう。

そこを目指したい。

### 自虐ブログは続ける

この自虐ブログは続けるつもりだ。

ユーザーが30人になっても、100人になっても、
「まだまだです」「全然です」「お恥ずかしい」
と言い続ける。

謙虚なのではない。
自虐がウケると分かってしまったからだ。

これをマーケティング戦略と呼ぶのか、ただの性格と呼ぶのかは、読者の判断に委ねる。
