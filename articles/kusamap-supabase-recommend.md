## Clawdbot旋風

2026年1月、開発者たちの間で話題をさらっているものがあります。

**Clawdbot**。

自己増殖するパーソナルAIアシスタント。ローカルで動く。革命的。

Xのタイムラインは Clawdbot の話題で埋め尽くされています。

「Clawdbot すごい」
「Clawdbot 最高」
「Clawdbot で人生変わった」

盛り上がっていますね。

**私は別の話をします。**

## AIはSupabaseが大好き

今日は、AIがやたらとSupabaseを薦めてくる問題について書きます。

Clawdbot？知りません。

私が向き合うべきは、Claude Codeとの日常です。

## 技術スタック相談あるある

Claude Codeに技術スタックを相談すると、こうなります。

私：「新しいWebアプリを作りたいんだけど、技術スタックどうしよう」

Claude Code：「バックエンドは**Supabase**がおすすめです」

私：「……」

Claude Code：「PostgreSQLベースで、認証機能も組み込まれていて、リアルタイム機能もあって——」

**知ってる。**

前回も聞いた。前々回も聞いた。その前も聞いた。

## 鉄板ネタとしてのSupabase

Claude Codeにとって、Supabaseは**鉄板ネタ**なのです。

お笑い芸人が持ちネタを繰り返すように、Claude CodeはSupabaseを繰り返す。

「データベースどうしよう」→ Supabase
「認証どうしよう」→ Supabase Auth
「ストレージどうしよう」→ Supabase Storage
「リアルタイム機能欲しい」→ Supabase Realtime

**全部Supabase。**

Supabaseの回し者かと疑いたくなるレベルです。

## Next.js 14問題の再来

以前、Claude CodeがNext.js 16を使っているのにNext.js 14を薦めてくる問題について書きました。

あれと同じ構造です。

AIには「推しの技術スタック」があるのです。

そして、その推しを布教することに躊躇がない。

「Supabase使いませんか？」
「Supabase便利ですよ？」
「Supabaseなら簡単にできますよ？」

宗教勧誘か。

## 他の鉄板ネタたち

Supabaseだけではありません。

Claude Codeの鉄板ネタをまとめてみましょう。

### Vercel

「デプロイどうしよう」

この質問に対する答えは、100%の確率で**Vercel**です。

AWS？GCP？Azure？

聞いてもいないのに「Vercelなら簡単にデプロイできますよ」と言ってくる。

Vercelの営業部門にAIがいるのかもしれません。

### Tailwind CSS

「スタイリングどうしよう」

**Tailwind CSS**一択です。

普通のCSS？CSS Modules？styled-components？

選択肢として提示されることすら稀。

「Tailwind CSSでユーティリティファーストなアプローチを——」

聞き飽きた。

### まとめると

Claude Codeの推し技術スタック：

- フレームワーク：Next.js（できれば14）
- バックエンド：Supabase
- デプロイ：Vercel
- スタイリング：Tailwind CSS

これ以外の組み合わせを提案されることは、ほぼありません。

## 草マップwの技術スタック

ここで告白します。

草マップwの技術スタック：

- フレームワーク：Next.js 16
- バックエンド：**Supabase**
- デプロイ：Vercel
- スタイリング：Tailwind CSS

**ほぼ完全一致**です。

私はClaude Codeの言いなりになっていました。

## 言い訳をさせてほしい

いや、Supabaseは本当に便利なんですよ。

PostgreSQLベースで、PostGISも使える。位置情報を扱う草マップwには最適でした。

認証機能も組み込まれていて、LINE認証の実装も比較的簡単でした。

Vercelも、Next.jsとの相性が抜群で——

……言い訳がClaude Codeの説明と同じになっている。

**洗脳されている。**

## AIの推しに従うべきか

真面目に考えてみましょう。

AIが特定の技術スタックを推すのには、理由があります。

1. **学習データに多い**: ドキュメントやチュートリアルが豊富な技術は、AIがよく知っている
2. **実績がある**: 多くの開発者が使っている = 成功事例が多い
3. **相性が良い**: Next.js + Supabase + Vercel は、実際に相性が良い

つまり、AIの推しは**ある程度合理的**なのです。

## でも、思考停止はよくない

とはいえ、AIの言いなりになるのは危険です。

プロジェクトによって最適な技術スタックは異なります。

- 大規模なら AWS や GCP の方が良いかもしれない
- レガシーシステムとの連携なら別の選択肢があるかもしれない
- チームのスキルセットによっては別の技術が適しているかもしれない

AIは「一般的なベストプラクティス」を提案しているだけ。

あなたのプロジェクトに最適かどうかは、**あなたが判断**すべきです。

## 私の場合

とはいえ、私は12人のユーザーしかいない個人開発サービスを運営しています。

大規模なインフラは不要。レガシーシステムとの連携もない。チームは私1人。

AIの推しで十分でした。

むしろ、AIの推しに従ったおかげで、6時間でサービスを立ち上げられました。

## 結論

Clawdbot で世界が変わるかもしれません。

でも私は、Claude Codeが Supabase を推してくる日常を生きています。

それはそれで、悪くない。
