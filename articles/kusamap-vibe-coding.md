# はじめに

こんにちは、フェニックスです。
今回は「草マップw」という位置情報アプリをバイブコーディングで開発した話をシェアします。

開発時間はわずか6時間。しかもコーディング自体は半分以下の時間で、残りは確認・検証作業でした。
AIと対話しながら開発を進める「バイブコーディング」の威力を実感した体験談です。

---

## 草マップwとは？

「草マップw」（Kusamap W）は位置情報を活用したWebアプリです。

「草」というのはインターネットミームで「笑う」という意味。
日本のネット文化では「www」が笑いを表し、それが草に見えることから「草」＝「笑い」という意味で使われています。

**公開URL**: [https://www.vanishing-vision.com/](https://www.vanishing-vision.com/)

### コンセプト：「この街に、草生やそうぜ。」

草マップwの核心は、**ソウルライクゲームのメッセージ機能を現実世界に実装した**ことにあります。

| ゲーム（エルデンリング等） | 草マップw |
|---------------------------|-----------|
| ダンジョンに先人のメッセージ | 街角に誰かの投稿 |
| 「この先、崖」の警告 | 「この店、激混み」の共有 |
| 評価で残る/消える | 「w」で延命/消滅 |
| 見知らぬ冒険者の助け合い | 見知らぬ街の人との繋がり |

現実世界を冒険するすべての人へ。
誰かが残した「草」を見つけ、価値があれば「w」を送る。
あなたの一言が、次に来る誰かの助けになる。

## 草マップwの主な特徴

### 1. エフェメラル（消える）メッセージ

草マップの根底にある価値観は「**情報の価値は鮮度に比例する**」です。

- 投稿は**24時間で自動消滅**
- 「w」リアクションで延命可能
- **100個以上の「w」**で永久保存（大草原）

位置情報と結びついた情報は、その瞬間・その場所でこそ意味を持つ。
昨日の「ここ混んでる」は今日のノイズでしかない。

| 従来のSNS | 草マップ |
|-----------|----------|
| 蓄積型（すべて残す） | 淘汰型（価値あるものだけ残る） |
| 古い情報が埋もれる | 古い情報は自動消滅 |
| ユーザーが取捨選択 | システムが鮮度で判断 |

### 2. 草の成長ライフサイクル

投稿された「草」は時間とともに状態が変化します。

| 時間 | 状態 | 表示 |
|------|------|------|
| 0-6h | 🌱 芽生え | 明るい緑 |
| 6-18h | 🌿 成長中 | 草色 |
| 18-24h | 🥀 枯れかけ | 黄緑 |
| 100+ w | 🌳 不滅 | 深緑 |

### 3. 位置情報制限

- **半径15m以内のみ閲覧可能**
- 実際にその場所に行かないと見えない
- 現実世界を「探索」する楽しさ

### 4. 「w」による価値の証明

消えゆく投稿に他者が価値を認めた時だけ延命。
**100w到達 = コミュニティが認めた普遍的価値**。

自己評価ではなく、**集合知による選別**です。

### 5. ゲーミフィケーション

- 1日の投稿上限（レベルで増加）
- 140文字制限
- 「延命の水」などの課金アイテム

### 6. LINE Mini App として動作

- LINE認証のみでログイン可能
- LINEユーザーが気軽に地域の「今」を共有できる

## バイブコーディングとは？

バイブコーディングは、AIに「委ねる」プログラミング手法です。

OpenAIの共同創業者であるアンドレイ・カルパシー氏が提唱した概念で、
コードの細部を自分で書くのではなく、AIに自然言語で指示を出して実装を委ねるスタイルです。

今回の開発では、Claude Codeを使ってこのバイブコーディングを実践しました。

---

## 技術スタック

### フロントエンド
- **Next.js 14** - Reactベースのフレームワーク
- **TypeScript** - 型安全な開発のため
- **Tailwind CSS** - 効率的なスタイリング
- **LINE Mini App（LIFF）** - LINEアプリ内で動作

### バックエンド
- **Supabase** - PostgreSQLベースのBaaS
  - データベース（PostGIS拡張で位置情報処理）
  - 認証
  - リアルタイム機能

### 認証・決済
- **LINE認証** - LINEアカウントでログイン
- **Stripe** - プレミアム機能の決済処理

### 開発ツール
- **Claude Code** - AIペアプログラミング
- **VS Code** - コードエディタ

## 開発の流れ

バイブコーディングでの開発は、従来のコーディングとは大きく異なります。

### 1. 要件をAIに伝える
まず、作りたいアプリの概要をClaude Codeに説明しました。
- 位置情報を扱うアプリであること
- LINEログインを使いたいこと
- Supabaseをバックエンドに使うこと

### 2. AIが提案・実装
Claude Codeが適切なファイル構成やコードを提案・生成してくれます。
TypeScriptの型定義からTailwind CSSのスタイリングまで、一貫して対応。

### 3. 確認・検証・修正
AIが生成したコードを確認し、動作検証を行います。
問題があれば、その内容をAIに伝えて修正を依頼。

このサイクルを繰り返すことで、アプリが完成していきます。

## 開発時間の内訳

総開発時間は**6時間**でしたが、内訳は興味深いものでした。

| 作業内容 | 時間 |
|---------|------|
| コーディング（AI生成含む） | 約2.5時間 |
| 確認・検証作業 | 約3.5時間 |

**コーディング自体は半分以下の時間**で済みました。

時間がかかったのは、むしろ確認・検証作業です。
- AIが生成したコードが正しく動作するか
- セキュリティ的に問題ないか
- UIが意図通りになっているか

これらをすべて検証しながら進めていく必要があります。

バイブコーディングは「コードを書かない」のではなく、
**「コードを書く時間を最小化し、検証と意思決定に集中する」**開発手法だと感じました。

## LINE連携について

今回のアプリでは、LINEとの連携を実装しました。

### LINE ログイン認証
日本のユーザーにとって、LINEアカウントでログインできるのは大きなメリットです。
新たにアカウントを作成する手間がなく、気軽に利用開始できます。

### LINE Messenger API
メッセージング機能との連携により、アプリの通知やコミュニケーション機能を強化。
位置情報と組み合わせることで、より便利な体験を提供できます。

SupabaseとLINE認証の連携設定は、Claude Codeの助けを借りてスムーズに実装できました。

## 苦労したこと

正直に言うと、コーディング自体で苦労したことは**特にありませんでした**。

Claude Codeが適切なコードを生成してくれるため、
従来のように「この書き方で合っているか？」「エラーの原因は何か？」と悩む時間が大幅に減りました。

ただし、**確認作業の多さ**は想定以上でした。
- 生成されたコードの動作確認
- エッジケースのテスト
- 各機能の連携確認

これらを丁寧に行う必要があり、結果として開発時間の半分以上を占めました。

---

## バイブコーディングのメリット・デメリット

### メリット
- **開発速度が圧倒的に速い** - 6時間でアプリが完成
- **技術的なハードルが下がる** - 細かい文法やAPIの使い方を覚えていなくてもOK
- **試行錯誤が容易** - 「こうしたい」と伝えるだけで別のアプローチを提案してもらえる

### デメリット
- **検証作業は省略できない** - AIが生成したコードを盲目的に信じるのは危険
- **要件を明確に伝える力が必要** - 曖昧な指示では期待通りの結果が得られない
- **デバッグ時の理解が浅くなりがち** - 自分で書いていないコードのデバッグは難しい場合がある

## 余談：ドメイン「Vanishing Vision」の由来

少し余談ですが、草マップwのドメイン「vanishing-vision.com」には深い意味があります。

### Xのインディーズアルバムから

「Vanishing Vision」は、日本のロックバンド**X（現X JAPAN）**のインディーズアルバムのタイトルです。

このタイトルには面白い文字遊びが隠されています：

- **V**anishing **V**ision の頭文字「V」と「V」
- VとVを**横に並べる**と → **W**
- VとVを**上下反転させて縦に並べる**と → **X**
- アルファベット順でも **V → W → X** と並んでいる

### アプリのコンセプトとの一致

「Vanishing Vision」は「消えていくイメージ」という意味ですが、これが草マップwのコンセプトと見事にマッチしていました。

**草マップwの仕組み：**
- 投稿は**24時間で消える**仕組み
- いいねの代わりに「**W**」を付けてもらえる
- **Wがたくさん付くと殿堂入り**し、24時間経っても消えない

つまり、残っている投稿は：
1. **新しい最新の情報**か
2. **多くの人に支持された優良な投稿**

のどちらかということになります。

消えていく情報の中で、本当に価値のあるものだけが残る。
この「Vanishing Vision」という言葉がアプリのコンセプトにぴったりだったので、このドメインを取得しました。
